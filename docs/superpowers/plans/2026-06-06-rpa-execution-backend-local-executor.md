# RPA Execution Backend And Local Executor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the RPA Web backend execution layer that can verify/run a stored `flow.hardened.py`, stream execution events/logs, support cancellation, and expose execution artifacts without adding RPA semantics to daemon core.

**Architecture:** `apps/rpa-local-web` owns all RPA execution logic. The daemon remains only a Claude Code runner and artifact source; it does not know `executionId`, Playwright, Python scripts, DSL steps, screenshots, or RPA storage. This slice implements backend APIs and local process execution only; React verification UI and `.rpa.zip` import/export remain separate later slices.

**Tech Stack:** TypeScript ESM, Express 5, Vitest, Node `child_process.spawn`, Node `fs/promises`, Node streams, existing RPA DSL/artifact contracts, local filesystem execution storage.

---

## Scope Boundary

This slice implements:

- RPA Web-owned execution storage under `RPA_LOCAL_STORAGE_ROOT`.
- `executionId`, `flowId`, and optional `daemonRunId` association.
- Execution API endpoints under `/api/rpa/executions`.
- Server-sent event stream for execution lifecycle, step, log, screenshot, artifact, and completion events.
- A local process runner that launches the generated Python/Playwright script with a stable CLI contract.
- Timeout, cancellation, stdout/stderr line capture, terminal status handling.
- Safe execution artifact collection from the execution directory.
- A fake-script test path that exercises success, failure, cancellation, log streaming, and artifact collection without requiring real Playwright.

This slice does not implement:

- React runtime verification UI components.
- Playwright codegen orchestration.
- Natural-language generation workflow.
- `.rpa.zip` import/export package routes.
- Real browser screenshot capture logic inside generated scripts.
- Trace/video policy beyond safe storage/list/download hooks.
- Any change under `apps/daemon/src/**`.

## Contract Decisions

- Execution IDs use prefix `exec_` and are generated by RPA Web, not daemon.
- Execution statuses: `queued | running | canceling | succeeded | failed | canceled | timed_out`.
- Execution modes: `verify | run`.
- Verify defaults: `dryRun: true`, `headless: false`.
- Run defaults: `dryRun: false`, `headless: true`.
- RPA Web writes runtime params to `run.params.json` inside the execution directory before spawning the script.
- Local script CLI shape:

```text
python3 flow.hardened.py --mode verify|run --params <executionDir>/run.params.json --execution-dir <executionDir> [--dry-run] [--headed|--headless]
```

- Tests may inject a fake command using `pythonCommand` and `pythonArgs` so CI does not depend on Python or Playwright.
- Execution events are JSON-safe and never include absolute daemon workspace paths.
- RPA Web storage may use absolute local paths internally, but API responses return `executionId`, `flowId`, artifact ids, file names, relative paths, sizes, hashes, and URLs only.
- RPA Web resolves `storageRoot` to an absolute path internally, relative to `process.cwd()` when configured as a relative path.
- Flow directory is always `<storageRoot>/flows/<flowId>` where `flowId` is validated by the existing `safeFlowId()` helper.
- Executor must reuse the existing `safeFlowId()` and `resolveFlowArtifactPath()` helpers from `apps/rpa-local-web/src/server/flow-store.ts` instead of rebuilding looser path rules.
- The executor reads and validates `flow.dsl.json` before creating an execution, derives `maskedParamIds` from `param.mask === true || param.type === 'secret'`, passes those ids into the store, and only then writes `execution.json`.
- `run.params.json` stores real runtime values for the local script. `execution.json`, SSE events, and `/logs` responses store only redacted summaries.
- stdout/stderr lines and executor error messages are sanitized before persistence and streaming. Minimal MVP redaction replaces the absolute `storageRoot` prefix with `[rpa-storage]`.
- Executor errors use structured `error.code`; `error.message` must not directly expose raw `fs` errors that include absolute local paths.
- SSE subscriptions are replayable: a late subscriber first receives all persisted `events.jsonl` records in order and then receives live events.
- SSE replay/live handoff must be race-safe: register a live buffer before replaying history, then flush buffered live events after history by event sequence while dropping duplicate sequence numbers.
- SSE responses close after the terminal `run.completed` event is sent.
- `RpaExecutionMode` and `RpaExecutionStatus` are defined once in `shared/rpa-api-types.ts`; internal executor types import or re-export those shared types.
- Current screenshot is represented as the latest collected artifact with role `screenshot`; the next UI slice will render it.
- `.rpa.zip` package export remains in the later runtime-params/import-export slice. This slice only downloads execution artifacts.
- This slice does not add `GET /api/rpa/executions` list/search. Keep store internals compatible with adding list by `flowId` or `daemonRunId` later.
- Large artifact hashing may start with simple file reads for MVP tests; switch to stream hashing before trace/video review bundles become large.

## Storage Layout

Default root:

```text
.rpa-local/
  flows/
    <flowId>/
      flow.dsl.json
      flow.hardened.py
      config.example.json
      parameterization-report.md
      hardening-report.md
  executions/
    <executionId>/
      execution.json
      run.params.json
      events.jsonl
      logs/
        stdout.log
        stderr.log
      artifacts/
        screenshots/
        downloads/
        trace/
        video/
```

`RPA_LOCAL_STORAGE_ROOT` can override `.rpa-local`. Add `.rpa-local/` to `.gitignore`.

## API Contract

Add these endpoints to RPA Web:

```text
POST /api/rpa/executions
GET  /api/rpa/executions/:executionId
POST /api/rpa/executions/:executionId/cancel
GET  /api/rpa/executions/:executionId/events
GET  /api/rpa/executions/:executionId/logs
GET  /api/rpa/executions/:executionId/screenshots/current
GET  /api/rpa/executions/:executionId/artifacts
GET  /api/rpa/executions/:executionId/artifacts/:artifactId/download
```

Start request:

```json
{
  "flowId": "case_query",
  "daemonRunId": "run_123",
  "mode": "verify",
  "dryRun": true,
  "headless": false,
  "timeoutMs": 30000,
  "params": {
    "case_no": "A123"
  }
}
```

Start response:

```json
{
  "executionId": "exec_abc123",
  "status": "queued",
  "flowId": "case_query",
  "daemonRunId": "run_123"
}
```

SSE event examples:

```text
event: run.started
data: {"type":"run.started","executionId":"exec_abc123","flowId":"case_query","mode":"verify","dryRun":true,"headless":false}

event: log
data: {"type":"log","executionId":"exec_abc123","stream":"stdout","message":"starting","timestamp":"2026-06-06T00:00:00.000Z"}

event: step.started
data: {"type":"step.started","executionId":"exec_abc123","stepId":"open_query","timestamp":"2026-06-06T00:00:01.000Z"}

event: artifact.created
data: {"type":"artifact.created","executionId":"exec_abc123","artifactId":"art_f3ab12cd34ef5678","role":"screenshot","relativePath":"artifacts/screenshots/open_query.png"}

event: run.completed
data: {"type":"run.completed","executionId":"exec_abc123","status":"succeeded","exitCode":0}
```

## Acceptance Checklist

- [ ] `RPA_LOCAL_STORAGE_ROOT` config exists and defaults to `.rpa-local`.
- [ ] Execution records preserve `executionId`, `flowId`, optional `daemonRunId`, mode, dryRun, headless, status, failedStepId, redacted params summary, and terminal errors.
- [ ] Runtime params are written to `run.params.json` inside execution storage.
- [ ] Local process runner uses `spawn` with `shell: false`.
- [ ] Timeout and cancellation produce terminal statuses.
- [ ] stdout/stderr are stored and streamed as log events.
- [ ] SSE emits `run.started`, `log`, `artifact.created`, and `run.completed`; step events are supported by type contract even if fake scripts do not emit all of them yet.
- [ ] Artifact collector safely lists/downloads files from the execution directory only.
- [ ] Current screenshot endpoint returns the latest screenshot artifact or 404 when absent.
- [ ] Execution route responses do not expose daemon workspace absolute paths or RPA Web storage absolute paths.
- [ ] Fake script tests cover success, failure, timeout, cancellation, and artifact collection.
- [ ] No code under `apps/daemon/src` imports or interprets RPA execution concepts.
